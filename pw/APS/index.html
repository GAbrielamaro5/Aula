Analise e Projeto de Sistemas



Análise e Projeto de Sistemas junho 25, 2017 Análise e projetos de sistemas fazem parte da engenharia de software. Ambas são disciplinas de modelagem, a análise antecede o projeto e deve certificar que conhece os requisitos funcionais e não funcionais do sistema. Requisitos funcionais e não funcionais Requisitos funcionais especificam as funcionalidades e serviços do sistemas. Ex.: Cadastrar um cliente, pesquisar um produto pelo nome, Requisitos não funcionais (atributos de qualidade) especificam as propriedades e restrições do sistema, precisa ser realizado para que o software atenda seu propósito. Os requisitos não funcionais podem ser mais críticos que os requisitos funcionais. Ex.: Compatibilidade com outras verões do sistema, O sistema deverá suportar uma carga máxima de 1000 usuários simultâneos, Apenas usuários com privilégio x poderão realizar operações y.
